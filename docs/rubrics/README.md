# Evaluation Rubrics

This directory contains the evaluation rubrics and criteria for the Automated Leave Management System project.

## Files

- **[evaluation-rubrics.md](evaluation-rubrics.md)** - Detailed evaluation rubrics with scoring criteria
- **[Software Engineering Project Implementation Phase Evaluation Rubric (1).pdf](../Software%20Engineering%20Project%20Implementation%20Phase%20Evaluation%20Rubric%20(1).pdf)** - Original rubric PDF document

## Evaluation Criteria Summary

The project is evaluated based on 8 main criteria:

1. **Requirements Analysis & Documentation** (20 points)
2. **System Design & Architecture** (20 points)
3. **Implementation & Code Quality** (25 points)
4. **Testing** (20 points)
5. **Documentation** (10 points)
6. **Deployment & DevOps** (10 points)
7. **Team Collaboration** (10 points)
8. **Project Management** (5 points)

**Total: 120 points**

## Key Requirements

### Code Quality Metrics
- ESLint errors: **0 allowed**
- ESLint warnings: **Maximum 10 allowed**
- Test coverage: **Minimum 75% required**
- Security vulnerabilities: **No critical vulnerabilities allowed**

### Documentation Requirements
- Requirements document
- Design document with UML diagrams
- API documentation
- User guide
- Developer guide
- README with setup instructions

### CI/CD Requirements
- Automated testing
- Code coverage reporting
- Linting
- Security scanning
- Deployment automation

### Team Requirements
- Regular commits with meaningful messages
- Code reviews for all changes
- Proper branching strategy
- Sprint planning and tracking

## Grading Scale

- **A+ (90-100%):** 108-120 points
- **A (80-89%):** 96-107 points
- **B+ (70-79%):** 84-95 points
- **B (60-69%):** 72-83 points
- **C (50-59%):** 60-71 points
- **F (<50%):** < 60 points

For detailed scoring criteria, please refer to [evaluation-rubrics.md](evaluation-rubrics.md).
